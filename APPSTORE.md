## Apple In-App Purchase Setup

The app uses direct StoreKit 2 integration (no RevenueCat). You need to configure products in App Store Connect and set up server notifications.

### Step 1: Create In-App Purchase Key

1. Go to **App Store Connect > Users and Access > Integrations > In-App Purchase**
2. Click **Generate In-App Purchase Key**
3. Give it a name (e.g. `BonistockIAPKey`)
4. **Download** the `.p8` file immediately (you can only download it once)
5. Note the **Key ID** (shown in the table after generating)
6. Note the **Issuer ID** (shown at the top of the Integrations page, UUID format like `f58049da-9ac0-4d5b-981e-dd4b689dcba3`)

### Step 2: Note Your Apple App ID

1. Go to **App Store Connect > Apps > Bonistock > App Information**
2. Under **General Information**, find **Apple ID** — it's a numeric ID (e.g. `6744571234`)

### Step 3: Create IAP Products

Go to **App Store Connect > Apps > Bonistock > Monetization > In-App Purchases**.

#### Subscriptions

Go to **Monetization > Subscriptions** and create a subscription group:

| Field | Value |
|-------|-------|
| Group Name | `Bonistock Plus` |
| Group Reference Name | `bonistock_plus` |

Then create two subscriptions within the group:

| Product ID | Reference Name | Duration | Price Tier |
|------------|---------------|----------|------------|
| `com.bonifatus.bonistock.plus.monthly` | Plus Monthly | 1 month | Choose tier (e.g. $6.99/mo) |
| `com.bonifatus.bonistock.plus.annual` | Plus Annual | 1 year | Choose tier (e.g. $49.99/yr) |

For each subscription:
- Add **Localization** (EN + DE) with display name and description
- Set a **Review Screenshot** (simple screenshot of the pricing page)
- Enable **Subscription Prices** for US and Germany

#### Non-Consumable IAPs (Day Passes)

Go to **Monetization > In-App Purchases** and create three non-consumable products:

| Product ID | Reference Name | Price Tier |
|------------|---------------|------------|
| `com.bonifatus.bonistock.pass.1day` | 1-Day Pass | Choose tier (e.g. $1.99) |
| `com.bonifatus.bonistock.pass.3day` | 3-Day Pass | Choose tier (e.g. $3.99) |
| `com.bonifatus.bonistock.pass.12day` | 12-Day Pass | Choose tier (e.g. $9.99) |

For each:
- Add **Localization** (EN + DE) with display name and description
- Set a **Review Screenshot**
- Apple prices should be ~30% higher than Stripe prices to offset Apple's commission

### Step 4: Configure Server Notifications V2

1. Go to **App Store Connect > Apps > Bonistock > App Information**
2. Scroll down to **App Store Server Notifications**
3. Set:
   - **Production Server URL**: `https://bonistock.com/api/apple/webhook`
   - **Sandbox Server URL**: `https://dev.bonistock.com/api/apple/webhook`
   - **Notification Version**: `Version 2 Notifications`

### Step 5: Deploy Docker Secrets

On the server, create the 4 secrets per environment. See `providers.md` section 9 for the exact commands.

Quick reference:
```bash
SSH="/c/Windows/System32/OpenSSH/ssh.exe root@159.69.180.183"

# Dev (remove stack first, create secrets, redeploy)
$SSH "docker stack rm bonistock-dev && sleep 5"
cat SubscriptionKey_XXXXXX.p8 | $SSH "docker secret rm bonistock_dev_APPLE_IAP_KEY_P8 2>/dev/null; docker secret create bonistock_dev_APPLE_IAP_KEY_P8 -"
echo -n 'YOUR_KEY_ID' | $SSH "docker secret rm bonistock_dev_APPLE_IAP_KEY_ID 2>/dev/null; docker secret create bonistock_dev_APPLE_IAP_KEY_ID -"
echo -n 'YOUR_ISSUER_ID' | $SSH "docker secret rm bonistock_dev_APPLE_IAP_ISSUER_ID 2>/dev/null; docker secret create bonistock_dev_APPLE_IAP_ISSUER_ID -"
echo -n 'YOUR_APPLE_APP_ID' | $SSH "docker secret rm bonistock_dev_APPLE_APP_ID 2>/dev/null; docker secret create bonistock_dev_APPLE_APP_ID -"
$SSH "cd /home/deploy/bonistock-dev && docker stack deploy -c docker-stack.dev.yml bonistock-dev"

# Repeat for prod with bonistock_prod_ prefix
```

### Step 6: Link Products in Admin Dashboard

After deploying, go to the Bonistock admin dashboard and ensure each product has the correct `appleProductId` set:

| Product | Apple Product ID |
|---------|-----------------|
| Plus Monthly | `com.bonifatus.bonistock.plus.monthly` |
| Plus Annual | `com.bonifatus.bonistock.plus.annual` |
| 1-Day Pass | `com.bonifatus.bonistock.pass.1day` |
| 3-Day Pass | `com.bonifatus.bonistock.pass.3day` |
| 12-Day Pass | `com.bonifatus.bonistock.pass.12day` |

### Step 7: Test in Sandbox

1. Create a **Sandbox Tester** account: App Store Connect > Users and Access > Sandbox > Testers
2. On your iOS device, go to Settings > App Store > Sandbox Account and sign in with the tester
3. Open the app, go to pricing, tap Buy/Subscribe
4. The native Apple payment sheet should appear with sandbox pricing
5. After purchase, check your server logs: `$SSH "docker service logs bonistock-dev_app --tail 50"`
6. Verify the subscription/pass was created in the database
